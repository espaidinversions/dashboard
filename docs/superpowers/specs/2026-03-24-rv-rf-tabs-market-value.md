# RV/RF Tabs Market Value Charts — Design Spec

**Date:** 2026-03-24
**Status:** Approved

---

## Goal

Enhance the Renda Variable and Renda Fixa sub-tabs in Mercats Públics with a market value over time chart per position; add the same chart to individual position detail pages; and fix a `globalSearch` state leak that silently corrupts capital calls filter counts when the user is on the Mercats Públics section.

## Architecture

Five independent deliverables, lowest to highest risk:

1. **New Python script** (`portfolio_export_js.py`) — reads the generated `portfolio_value.csv`, aggregates to monthly data points, writes `src/data/portfolioValues.js`.
2. **Committed placeholder + generated data file** (`src/data/portfolioValues.js`) — a placeholder `export const PM_VALUES = {};` is committed so the import never crashes; the Python script overwrites it with real data when run locally. The file is **not** gitignored.
3. **Shared helper** (`fmtMonth` added to `src/utils.js`) — Catalan month abbreviation formatter, imported by both PMTipusTab and PMPositionDetail.
4. **PMTipusTab enhancement** — replace Chart 2 with market-value-over-time; add Brut/Net TER toggle.
5. **PMPositionDetail enhancement** — add market value over time chart card.
6. **Dashboard bug fix** — scope `globalSearch` capital-calls filtering to alternatives section only.

## Tech Stack

React 18, Recharts (LineChart, ResponsiveContainer), existing `useTheme()` hook, Python 3 + pandas.

## Data Layer

### `scripts/portfolio_export_js.py` (new)

- Reads `Mercats Públics/portfolio_value.csv` (columns: `date, isin, nom, custodian, units, nav, value_eur`)
- Groups rows by `(isin, custodian)`; within each group keeps the **last row per calendar month** (truncate `date` to `YYYY-MM`, keep latest date in that month)
- Outputs `src/data/portfolioValues.js`:

```js
// Auto-generated by scripts/portfolio_export_js.py — do not edit manually
export const PM_VALUES = {
  "IE00B1G3DH73": {
    "Abel Font": [
      { "date": "2023-01", "value": 123456 },
      ...
    ]
  },
  ...
};
```

- Run after `portfolio_build_values.py`

### `src/data/portfolioValues.js` (committed placeholder)

Commit this file as the starting state; the Python script overwrites it when run:

```js
// Auto-generated by scripts/portfolio_export_js.py — do not edit manually
// Run scripts/portfolio_export_js.py to populate with real data
export const PM_VALUES = {};
```

This ensures the app builds and runs correctly even without local price data.

## Shared Helper — `src/utils.js`

Add `fmtMonth` as a named export (used by both PMTipusTab and PMPositionDetail):

```js
const _CAT_MONTHS = ["","Gen","Feb","Mar","Abr","Mai","Jun","Jul","Ago","Set","Oct","Nov","Des"];
export function fmtMonth(yyyymm) {
  const [y, m] = yyyymm.split("-");
  return `${_CAT_MONTHS[+m]} '${y.slice(2)}`;
}
```

## PMTipusTab Changes

### Chart 2 Replacement: Market Value Over Time

PMTipusTab has two charts:
- **Chart 1** (portfolio weighted return bar chart) — **kept unchanged**
- **Chart 2** (top-N individual positions annual returns line chart) — **replaced** with the market value chart below

**New Chart 2: Market Value Over Time**

- **Component**: `LineChart` inside `ResponsiveContainer` (height 280)
- **Import**: add `{ PM_VALUES }` from `"../data/portfolioValues"` and `{ fmtMonth }` from `"../utils"`
- **Data construction** (one series per filtered position, merged across custodians):

```js
const chartData = useMemo(() => {
  // Collect all dates across all positions
  const dateSet = new Set();
  positions.forEach(pos => {
    const custodians = PM_VALUES[pos.isin];
    if (!custodians) return;
    Object.values(custodians).forEach(series => series.forEach(d => dateSet.add(d.date)));
  });
  const dates = [...dateSet].sort();

  return dates.map(date => {
    const row = { date };
    // Top 12 positions by valorMercat (sorted before this useMemo)
    top12.forEach(pos => {
      const custodians = PM_VALUES[pos.isin];
      if (!custodians) { row[pos.isin] = null; return; }
      row[pos.isin] = Object.values(custodians)
        .map(s => s.find(d => d.date === date)?.value ?? 0)
        .reduce((a, b) => a + b, 0) || null;
    });
    return row;
  });
}, [positions, top12]);
```

- **Cap**: top **12** positions by `valorMercat` (matches `PM_COLORS` palette length exactly); palette cycles if needed but cap of 12 avoids it
- **X-axis**: `fmtMonth(date)`
- **Y-axis**: `fmtM` (already imported in PMTipusTab)
- **Tooltip**: position name + `fmtM(value)`
- **Fallback**: if `Object.keys(PM_VALUES).length === 0`, do not render the chart card

### Brut / Net TER Toggle

- Pill toggle right-aligned, same row as the existing Tots/Directe/Bankinter pills
- Two states: `"brut"` (default) and `"net"` — `useState("brut")`, not persisted
- The toggle controls **three places** in PMTipusTab:

**1. Chart 1 (`portfolioData` useMemo) — currently always uses `netRend`**
- `"brut"` mode: compute weighted average using `p[field]` (raw value) instead of `netRend(p, field)`
- `"net"` mode: keep existing behaviour, `netRend(p, field)`

**2. Table cell rendering — currently always renders `<PctChip v={netRend(p, field)} />`**
- `"brut"` mode: render `<PctChip v={p[field]} />` for each annual column
- `"net"` mode: keep existing `<PctChip v={netRend(p, field)} />`

**3. CAGR column**
- `"brut"` mode: `cagr(p.rendInici, yearsHeld(p))`
- `"net"` mode: `cagr(netRendInici(p), yearsHeld(p))` (existing behaviour)

Market value chart is **unaffected** by this toggle.

## Dashboard Bug Fix

### `globalSearch` leaking into capital calls `filtered`

**Location**: `Dashboard.jsx`, the `filtered` useMemo (around line 249–264).

**Root cause**: `globalSearch` filters the capital-calls `filtered` array regardless of which section is active. When the user types in the search bar on Mercats Públics, `filtered` is silently narrowed. On returning to Alternatives, the KPI counts (filtered.length, fCalls, fDist) are stale until the search is cleared.

**Scope of fix**: only the `filtered` useMemo. `globalSearch` is intentionally global for other consumers (`SearchersTab`, `PortfolioCompaniesTab`, `FundsIndexInner`, `CompaniesIndexInner`) — do not touch those or the `globalSearch` state itself.

**Fix** — wrap only the capital-calls clause:

```js
// Before (buggy):
if (globalSearch.trim()) {
  const q = globalSearch.trim().toLowerCase();
  d = d.filter(r => (r.fons || "").toLowerCase().includes(q));
}

// After (fixed):
if (section === "alternatives" && globalSearch.trim()) {
  const q = globalSearch.trim().toLowerCase();
  d = d.filter(r => (r.fons || "").toLowerCase().includes(q));
}
```

`section` is derived at line 410 as `(tab==="mercats-publics"||tab==="real-estate") ? tab : "alternatives"` — already in scope, no new state needed.

**Important — hoist `section`**: `section` is currently derived at line 410, which is after the `filtered` useMemo at line 249. Because the `section` constant must appear in the useMemo dependency array (and because JavaScript `const` has temporal dead zone semantics), `section` must be **moved above** the `filtered` useMemo. Cut the line:

```js
const section = (tab==="mercats-publics"||tab==="real-estate") ? tab : "alternatives";
```

and paste it immediately before the `filtered` useMemo. Then add `section` to the dependency array: `[baseTx, fFy, fVcpe, fEst, fCat, txSearch, globalSearch, section]`.
